## Why It Matters

Agentforce represents Salesforce's shift from the copilot model (assistive AI) to the agent model (autonomous AI). Where copilots suggest actions and wait for user confirmation, Agentforce agents can execute multi-step workflows independently — handling customer inquiries, updating records, and triggering processes.

For architects, the key question is where agents fit in your automation stack. They're not replacing Flows or Apex — they're sitting on top of them, orchestrating existing capabilities through natural language. Understanding the agent trust layer, grounding with Data Cloud, and guardrail configuration is essential.

## Alternatives

- **Agentforce Assistant** — assistive mode (formerly Einstein Copilot), user-in-the-loop
- **Prompt Builder** — template-based AI, used as agent actions within Agentforce
